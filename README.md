# mvc-practice (로그인 시스템)

웹 애플리케이션의 핵심인 로그인 유지와 미인증 접근 차단 기능을 구현한 프로젝트입니다.

## Tech Stack
- **Language:** Java
- **Framework:** Spring Boot, Spring MVC

## Key Features
- 사용자 회원가입 및 세션 기반 로그인/로그아웃 구현
- 인증되지 않은 사용자의 특정 페이지 접근 차단
  
## Technical Decisions & Troubleshooting

### 1. 보안을 고려한 Session 기반 로그인
- **문제:** 초기에는 로그인 상태를 유지하기 위해 쿠키만 사용했으나, 클라이언트가 쿠키 값을 임의로 변조하거나 탈취당할 경우 심각한 보안상의 문제가 발생했습니다.
- **해결:** 중요한 사용자 정보는 서버의 메모리에 안전하게 저장하고, 클라이언트에게는 추적 불가능한 Session ID만 쿠키로 전달하는 **세션 기반 인증 방식**을 사용했습니다.

### 2. Spring Interceptor를 활용한 공통 로직 처리
- **문제:** 로그인하지 않은 사용자가 URL을 직접 입력해 로그인이 필요한 페이지에 접근하는 것을 막아야 했습니다. 하지만 모든 컨트롤러 메서드에 로그인 확인 코드를 중복 작성하는 것은 비효율적입니다.
- **해결:** Spring의 **Interceptor**를 도입하여, 세션 유무를 검증하는 로직을 공통화해 유지 보수성을 높였습니다.
